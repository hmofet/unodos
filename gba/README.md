# UnoDOS / Nintendo Game Boy Advance (ARM7TDMI)

The **fifth** fresh contract-driven port and the **first on ARM** — a genuinely new
32-bit RISC world (a new `arm` / GNU-as unogen dialect). It consumes `gen/arm/` +
`[world.gba]`. The GBA hardware exceeds the minimal profile, but this is a MINIMAL
UnoDOS instance (CONTRACT-ARCH §9): one full-screen app at a time, directional nav.

Unlike the tile consoles, the GBA port draws into a flat **Mode 3 framebuffer**
(240×160, 16bpp BGR555) — no hardware tiles. It plots an 8×8 font and 16×16 icons
pixel by pixel, each pixel's palette **index** looked up in a 16-entry BGR555 table
in IWRAM, so the Theme app recolours by swapping the table and redrawing. It's the
software-framebuffer cousin of the PS2/Dreamcast ports, on a handheld.

## Status — M1 · M2 · M3 all shipped ✅

Headlessly verified by running the real ARM ROM on a **Unicorn ARM7TDMI** core
(`gba/harness.py`) and rendering the Mode 3 framebuffer — the same approach the C64/
Apple II ports use (py65 6502) and the MacPlus port uses (Unicorn 68K). The AUTOTEST
ROMs drive the pad through the same input path; nothing is faked (`build/*.png`):

- **M1 — launcher** (`build/desktop.png`): ARM bring-up from scratch, a 4-column
  colour icon grid (16×16 icons from the x86 donors) + the inverted "UnoDOS 3" bar.
- **M2 — navigation** (`build/nav.png`): REG_KEYINPUT, a VCOUNT-polled frame loop, a
  d-pad selection highlight (the selected label inverts), A launches, B returns.
- **M3 — apps** (`build/{app,clock,dostris,theme,music}.png`): SysInfo, live Clock
  (HH:MM:SS off the frame counter), Notepad, Files, Theme (cycles the BGR555 palette
  live), Music (the GBA square channel), and **Dostris** — the falling-blocks game,
  full colour. Tracker / OutLast / Pac-Man / Paint open framed placeholders.

`mGBA` also **loads** the ROM (its window title shows the internal title), but mGBA's
deferred-OpenGL display does not render to a focus-independent grab under headless
RDP, so the framebuffer screenshots come from the Unicorn harness. `gba/run.ps1`
still launches the ROM in mGBA for an on-display / audio ear-check on a real session.

## Hardware brought up (from scratch, ARM assembly)

| Part | Detail |
|---|---|
| CPU | **ARM7TDMI** @ 16.78 MHz, ARM (32-bit) mode; reset via a `b` at ROM offset 0 |
| Video | **Mode 3** (REG_DISPCNT `$0403`): 240×160 16bpp framebuffer at `$06000000`, BGR555 |
| Colour | a 16-entry BGR555 palette in IWRAM; pixels are indices looked up at plot time |
| Input | REG_KEYINPUT (`$04000130`), 10 keys, active-low |
| Timing | poll REG_VCOUNT (`$04000006`) for vblank (≥160) — no interrupts |
| Audio | square channel 1 (`$04000060`..`$04000064` + SOUNDCNT) |
| RAM | IWRAM `$03000000` (32 KB) — stack + variables; the framebuffer is VRAM |
| ROM | a plain cartridge image at `$08000000` (header logo left zeroed; mGBA boots it) |

## Build & run

```sh
sh gba/build.sh                                       # -> gba/build/unodos.gba
sh gba/build.sh nav|app|clock|theme|music|dostris     # AUTOTEST builds
python gba/harness.py gba/build/unodos.gba gba/build/desktop.png   # render via Unicorn
powershell -File gba/run.ps1 -Rom build\unodos.gba                 # load in mGBA (on-display)
```

- `mkdata.py` converts the shared font, the x86 icon donors (→ 16×16 index maps), the
  Dostris piece tables, the tune, and the BGR555 theme palettes into GNU-as data
  (`build/gfx.s`) plus a small `build/gfxequ.inc` of `.equ` constants the kernel needs
  at assemble time.
- `build.sh` runs mkdata, then assembles + links via the WSL `arm-none-eabi` toolchain
  (`as`/`ld`/`objcopy`) and fixes the ROM header checksum (`gbafix.py`). AUTOTEST is
  selected with `--defsym AUTOTEST=1 --defsym AT_*=1`.
- Source: `kernel.s` (boot, framebuffer + input + nav + render dispatch), `apps.inc.s`
  (app draws, clock/theme/music, AUTOTEST + strings), `dostris.inc.s` (the game).

## Toolchain
- **arm-none-eabi** binutils in WSL (`gcc-arm-none-eabi` package) — the ARM assembler/linker.
- **Unicorn Engine** (Python, `unicorn` 2.x) — the headless ARM core for screenshots.
- **mGBA 0.10.5** (`~/gba-tools/`, portable) — loads the ROM for an on-display check.

## Why ARM matters here
Every prior UnoDOS port is an 8/16-bit CISC (x86, 68K, 65xx, Z80, SM83). The GBA is
the first **32-bit RISC** target — it proves the Contract + the `minimal` profile
scale onto a fundamentally different ISA, with the boundary (`gen/arm/`) generated by
the same unogen as every other world.
