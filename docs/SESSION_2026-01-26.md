## Technical Details

### INT 0x80 Mechanism (verified working)
1. App calls `int 0x80` with AH = function index
2. Handler saves caller_ds and caller_es
3. Handler sets DS = 0x1000 (kernel)
4. Function dispatched via jump table
5. Functions use caller_ds/caller_es for app data access
6. Returns via IRET

### CGA Video Memory Layout (Mode 04h)
- Segment: 0xB800
- Even scanlines: offset 0x0000 - 0x1F3F
- Odd scanlines: offset 0x2000 - 0x3F3F
- 80 bytes per scanline (320 pixels / 4 pixels per byte)
- Y offset formula: (Y/2) * 80
