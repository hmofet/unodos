UnoDOS pc64

A GUI-first operating system that boots bare-metal on any 64-bit UEFI PC — straight into a themed desktop, with its own window manager, web browser, TCP/IP + TLS stack, and a 3D engine. No BIOS, no DOS, no runtime. This manual covers the pc64 world.

Get started › Tour the desktop

UnoDOS is a whole family of GUI-first operating systems — its original kernel is hand-written IBM PC/XT assembly — that now spans 20+ hardware targets. pc64 is the modern-PC member: the first 64-bit x86 world, written in C, that runs as a UEFI application on essentially every PC built since ~2007. It has been verified under QEMU + OVMF and on a Lenovo ThinkPad X1 Carbon Gen 8 booting from a USB stick.

What you get

A real desktop

Window manager, Start menu, taskbar, desktop icons, draggable/resizable windows — all keyboard- or pointer-driven.

10 live themes

A modern Aurora look (light/dark) plus eight retro skins, switchable live from the Control Panel.

Applications

Editor, Files, System, Clock, a Canvas demo, plus Paint, Music, a Tracker, three games and a 3D runner.

A web browser

Renders HTML, Markdown and CSS, runs JavaScript, and loads pages over HTTP and HTTPS.

Networking + TLS

A native e1000 driver, a from-scratch TCP/IP stack, DHCP/DNS, and CA-validated TLS 1.2.

Drivers

UEFI GOP framebuffer, keyboard & pointer, PC-speaker audio, PCI, FAT via the firmware, and xHCI USB.

Firmware-as-BIOS

pc64 never leaves the firmware behind at boot. UEFI's GOP hands the kernel a linear 32-bit framebuffer; Simple Text Input hands it the keyboard with modifier state; the Simple/Absolute Pointer protocols hand it a mouse where the firmware binds one; and the UEFI runtime services provide the real-time clock and shutdown/restart. Boot services stay alive — UEFI plays exactly the role that INT 10h/13h/15h play for the classic 16-bit kernel. Native drivers (xHCI, e1000) are the driver tail, not a bring-up requirement.

Which build this coversThis manual documents the default ./build.sh image — the unoui desktop shell. A separate legacy build keeps the earlier icon-desktop and is only retained as reference; everything it did now runs in the default shell.
