## Build System Changes

The build system now auto-generates version strings:

```makefile
# Generate build info include file
$(BUILD_INFO): BUILD_NUMBER VERSION
    @echo "; Auto-generated build info - DO NOT EDIT" > $@
    @echo "BUILD_NUMBER_STR: db 'Build: $(shell printf '%03d' $(BUILD_NUMBER))', 0" >> $@
    @echo "VERSION_STR: db 'UnoDOS v$(shell cat VERSION)', 0" >> $@
```

Usage:
- `make floppy144` - Build with current build number
- `make bump-build` - Increment build number for next build
- Build number is read from `BUILD_NUMBER` file
- Version is read from `VERSION` file

## Lessons Learned

1. **Real hardware differs from QEMU**: Register corruption only manifested on real hardware, not in emulator
2. **FAT filename conventions**: The 8.3 format in directory entries (space-padded, no dot) differs from the filename format expected by open functions (with dot separator)
3. **Dynamic configuration**: Build numbers should come from files, not hardcoded strings
